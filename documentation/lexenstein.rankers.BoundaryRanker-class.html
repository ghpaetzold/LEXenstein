getRankings(self, victor_corpus)

source code 

Ranks candidates with respect to their simplicity. Requires for the trainRanker function to be previously called so that a model can be trained.

Parameters:
  • victor_corpus - Path to a testing corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
Returns:
A list of ranked candidates for each instance in the VICTOR corpus, from simplest to most complex.

trainRanker(self, victor_corpus, positive_range, loss, penalty, alpha, l1_ratio, epsilon, k='all')

source code 

Trains a Boundary Ranker according to the parameters provided.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • loss - Loss function to be used. Values available: hinge, log, modified_huber, squared_hinge, perceptron.
  • penalty - Regularization term to be used. Values available: l2, l1, elasticnet.
  • alpha - Constant that multiplies the regularization term. Recommended values: 0.0001, 0.001, 0.01, 0.1
  • l1_ratio - Elastic net mixing parameter. Recommended values: 0.05, 0.10, 0.15
  • epsilon - Acceptable error margin. Recommended values: 0.0001, 0.001
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.

trainRankerWithCrossValidation(self, victor_corpus, positive_range, folds, test_size, losses=['hinge', 'modified_huber'], penalties=['elasticnet'], alphas=[0.0001, 0.001, 0.01], l1_ratios=[0.0, 0.15, 0.25, 0.5, 0.75, 1.0], k='all')

source code 

Trains a Boundary Ranker while maximizing hyper-parameters through cross-validation. It uses the TRank-at-1 as an optimization metric.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • folds - Number of folds to be used in cross-validation.
  • test_size - Percentage of the dataset to be used in testing. Recommended values: 0.2, 0.25, 0.33
  • losses - Loss functions to be considered. Values available: hinge, log, modified_huber, squared_hinge, perceptron.
  • penalties - Regularization terms to be considered. Values available: l2, l1, elasticnet.
  • alphas - Constants that multiplies the regularization term. Recommended values: 0.0001, 0.001, 0.01, 0.1
  • l1_ratios - Elastic net mixing parameters. Recommended values: 0.05, 0.10, 0.15
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.
