source code 

Creates a PLUMBErr error categorizer. This class implements the strategy introduced in: Paetzold, G. H.; Specia, L. PLUMBErr: An Automatic Error Identification Framework for Lexical Simplification. Proceedings of the 1st QATS. 2016. One can download BenchLS (dataset) and NNSVocab (complex) from http://ghpaetzold.github.io/data/PLUMBErr.zip

Parameters:
  • dataset - Path to a data in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • complex - Path to a file containing complex words. Each line of the file must contain a single word.

cumulativeAnalysis(self, identified, selected, ranked)

source code 

Performs the cumulative error identification analysis of a simplifier. In a cumulative analysis, the errors made during Complex Word Identification are carried onto the following steps of the pipeline.

Parameters:
  • identified - A vector containing one binary value (0 for simple, 1 for complex) for each word in the dataset. To produce the vector, one can run a Complex Word Identification approach from LEXenstein over the dataset.
  • selected - A vector containing the candidates selected for each instance in the dataset. To produce the vector, one can pair a Substitution Generation and a Substitution Selection approach from LEXenstein.
  • ranked - A vector containing the selected candidates ranked in order of simplicity. To produce the vector, one can run a Substitution Ranking approach from LEXenstein over the selected candidates provided.

nonCumulativeAnalysis(self, identified, selected, ranked)

source code 

Performs the non-cumulative error identification analysis of a simplifier. In a non-cumulative analysis, the errors made during Complex Word Identification are not carried onto the following steps of the pipeline.

Parameters:
  • identified - A vector containing one binary value (0 for simple, 1 for complex) for each word in the dataset. To produce the vector, one can run a Complex Word Identification approach from LEXenstein over the dataset.
  • selected - A vector containing the candidates selected for each instance in the dataset. To produce the vector, one can pair a Substitution Generation and a Substitution Selection approach from LEXenstein.
  • ranked - A vector containing the selected candidates ranked in order of simplicity. To produce the vector, one can run a Substitution Ranking approach from LEXenstein over the selected candidates provided.
Returns:
A report vector containing the errors made in each instance of the dataset, as well as a map containing total error counts for the entire dataset.
