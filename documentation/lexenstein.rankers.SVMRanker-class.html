getFeaturesFile(self, victor_corpus, output_file)

source code 

Creates a file containing feature values in SVM-Rank format. Produces the "features_file" parameter for functions getTrainingModel, getScoresFile and getRankings.

Parameters:
  • victor_corpus - Path to a corpus in the VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • output_file - Path in which to save the resulting feature values.

getRankings(self, features_file, scores_file)

source code 

Produces ranking scores in SVM-Rank format. The scores file produced can be used as the "scores_file" parameter of the getRankings function.

Parameters:
  • features_file - Path to features file produced over a testing VICTOR corpus. Should be produced by the getFeaturesFile function.
  • scores_file - Path to a scores file in SVM-Rank format. Should be produced by the getScoresFile function.
Returns:
A list of ranked candidates, from simplest to most complex.

getScoresFile(self, features_file, model_file, output_file)

source code 

Produces ranking scores in SVM-Rank format. The scores file produced can be used as the "scores_file" parameter of the getRankings function.

Parameters:
  • features_file - Path to features file produced over a testing VICTOR corpus. Should be produced by the getFeaturesFile function.
  • model_file - Path to a trained model file in SVM-Rank format. Should be produced by the getTrainingModel function.
  • output_file - Path in which to save the resulting ranking scores in SVM-Rank format.

getTrainingModel(self, features_file, c, epsilon, kernel, output_file)

source code 

Trains an SVM-Rank ranking model. The model produced can be used as the "model_file" parameter of the getScoresFile function.

Parameters:
  • features_file - Path to features file produced over a training VICTOR corpus. Should be produced by the getFeaturesFile function.
  • c - Trade-off between training error and margin. Recommended values: 0.001, 0.01
  • epsilon - Acceptable error margin. Recommended values: 0.00001, 0.0001
  • kernel - ID for the kernel to be used. Kernels available: 0 - Linear 1 - Polynomial 2 - Radial Basis Function 3 - Sigmoid
  • output_file - Path in which to save the resulting SVM-Rank model.
