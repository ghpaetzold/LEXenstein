addNgramCountsFileToShelve(ngrams_file, model_file)

source code 

Adds a n-gram counts file to an either new, or existing shelve dictionary. The shelve file can then be used for the calculation of several features. The file must be in the format produced by the "-write" option of SRILM ngram-count application.

Parameters:
  • ngrams_file - File containing n-gram counts.
  • model_file - Shelve file in which to save the n-gram counts file.

addTargetAsFirstToVictorCorpus(self, victor_corpus, output)

source code 

Creates a modified version of an input VICTOR corpus in which the target complex word is ranked first. Can be very useful for the training of Substitution Selection Models

Parameters:
  • victor_corpus - Path to a corpus in the VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • output - Path in which to save the modified VICTOR corpus.

addTranslationProbabilitiesFileToShelve(transprob_file, model_file)

source code 

Adds a translation probabilities file to an either new, or existing shelve dictionary. The shelve file can then be used for the calculation of features. To produce the translation probabilities file, first run the following command through fast_align: fast_align -i <parallel_data> -v -d -o <transprob_file>

Parameters:
  • transprob_file - File containing translation probabilities.
  • model_file - Shelve file in which to save the translation probabilities.

createConditionalProbabilityModel(folder, fileids, model, sep='/', encoding='utf8')

source code 

Creates an tagging probability model to be used along with the FeatureEstimator object. Files of tagged data must contain one sentence per line, and each line must follow the following format: <word_1><separator><tag_1> <word_2><separator><tag_2> ... <word_n-1><separator><tag_n-1> <word_n><separator><tag_n>

Parameters:
  • folder - Folder containing files of tagged sentences.
  • fileids - A list or regular expressions specifying the file names with tagged data in "folder".
  • model - File in which to save the trained model.
  • sep - Separator between words and tags in the files with tagged data.
  • encoding - Encoding of the files with tagged data.

createTaggedNgramsFile(ngrams_file, tagged_ngrams_file)

source code 

Creates a tagged version of an annotated n-gram counts file.

Parameters:
  • ngrams_file - File containing POS tag annotated n-gram counts. The file must be in the format produced by the "-write" option of SRILM. Each word in the corpus used must be in the following format: <word>|||<tag>
  • tagged_ngrams_file - File with tagged n-gram counts.

dependencyParseSentences(parser, sentences)

source code 

Use StanfordParser to parse multiple sentences. Takes multiple sentences as a list where each sentence is a list of words. Each sentence will be automatically tagged with this StanfordParser instance's tagger. If whitespaces exists inside a token, then the token will be treated as separate tokens. This method is an adaptation of the code provided by NLTK.

Parameters:
  • parser - An instance of the nltk.parse.stanford.StanfordParser class.
  • sentences - Input sentences to parse. Each sentence must be a list of tokens. @return A list of the dependency links of each sentence. Each dependency link is composed by the relation type, the source word, its position in the sentence, the target word, and its position in the sentence.

fitTranslationProbabilityFileToCorpus(translation_probabilities, corpus, output)

source code 

Creates a translation probabilities file that has only translations pertaining to the target complex words of a given VICTOR or CWICTOR corpus.

Parameters:
  • translation_probabilities - Path to a file containing the translation probabilities. The file must produced by the following command through fast_align: fast_align -i <parallel_data> -v -d -o <translation_probabilities>
  • corpus - Path to a corpus in the VICTOR or CWICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • output - Path in which to save the filtered translation probabilities file.

getVocabularyFromDataset(dataset, vocab_file, leftw, rightw, format='victor')

source code 

Extracts the vocabulary from a dataset in VICTOR or CWICTOR format. This vocabularies can be used along with SRILM in order for smaller n-gram count files to be produced.

Parameters:
  • dataset - Dataset from which to extract the vocabulary.
  • vocab_file - File in which to save the vocabulary.
  • leftw - Window to consider from the left of the target word.
  • rightw - Window to consider from the right of the target word.
  • format - Format of the dataset. Values accepted: victor, cwictor
