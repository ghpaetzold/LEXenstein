getRankings(self, victor_corpus)

source code 

Ranks candidates with respect to their simplicity. Requires for the trainRanker function to be previously called so that a model can be trained.

Parameters:
  • victor_corpus - Path to a testing corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
Returns:
A list of ranked candidates for each instance in the VICTOR corpus, from simplest to most complex.

trainRanker(self, victor_corpus, positive_range, C, kernel, degree, gamma, coef0, k='all')

source code 

Trains a SVM Boundary Ranker according to the parameters provided.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • C - Penalty parameter. Recommended values: 0.1, 1, 10.
  • kernel - Kernel function to be used. Supported values: 'linear', 'poly', 'rbf', 'sigmoid'.
  • degree - Degree of the polynomial kernel. Recommended values: 2, 3.
  • gamma - Kernel coefficient. Recommended values: 0.01, 0.1, 1.
  • coef0 - Independent term value. Recommended values: 0, 1.
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.

trainRankerWithCrossValidation(self, victor_corpus, positive_range, folds, test_size, Cs=[0.1, 1, 10], kernels=['linear', 'rbf', 'poly', 'sigmoid'], degrees=[2], gammas=[0.01, 0.1, 1], coef0s=[0, 1], k='all')

source code 

Trains a SVM Boundary Ranker while maximizing hyper-parameters through cross-validation. It uses the TRank-at-1 as an optimization metric.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • folds - Number of folds to be used in cross-validation.
  • test_size - Percentage of the dataset to be used in testing. Recommended values: 0.2, 0.25, 0.33
  • Cs - Penalty parameters. Recommended values: 0.1, 1, 10.
  • kernels - Kernel functions to be used. Supported values: 'linear', 'poly', 'rbf', 'sigmoid'.
  • degrees - Degrees of the polynomial kernel. Recommended values: 2, 3.
  • gammas - Kernel coefficients. Recommended values: 0.01, 0.1, 1.
  • coef0s - Independent term values. Recommended values: 0, 1.
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.
