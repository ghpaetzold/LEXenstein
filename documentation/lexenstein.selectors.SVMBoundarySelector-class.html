selectCandidates(self, substitutions, victor_corpus, temp_file, proportion, proportion_type='percentage')

source code 

Selects which candidates can replace the target complex words in each instance of a VICTOR corpus.

Parameters:
  • substitutions - Candidate substitutions to be filtered. It can be in two formats: A dictionary produced by a Substitution Generator linking complex words to a set of candidate substitutions. Example: substitutions['perched'] = {'sat', 'roosted'} A list of candidate substitutions selected for the "victor_corpus" dataset by a Substitution Selector. Example: [['sat', 'roosted'], ['easy', 'uncomplicated']]
  • victor_corpus - Path to a corpus in the VICTOR format. For more information about the file's format, refer to the LEXenstein Manual. User must have the privilege to delete such file without administrator privileges.
  • temp_file - File in which to save a temporary victor corpus. The file is removed after the algorithm is concluded.
  • proportion - Proportion of substitutions to keep. If proportion_type is set to "percentage", then this parameter must be a floating point number between 0 and 1. If proportion_type is set to "integer", then this parameter must be an integer number.
  • proportion_type - Type of proportion to be kept. Values supported: percentage, integer.
Returns:
Returns a vector of size N, containing a set of selected substitutions for each instance in the VICTOR corpus.

trainSelector(self, victor_corpus, positive_range, C, kernel, degree, gamma, coef0, k='all')

source code 

Trains a Boundary Ranker according to the parameters provided.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • C - Penalty parameter. Recommended values: 0.1, 1, 10.
  • kernel - Kernel function to be used. Supported values: 'linear', 'poly', 'rbf', 'sigmoid'.
  • degree - Degree of the polynomial kernel. Recommended values: 2, 3.
  • gamma - Kernel coefficient. Recommended values: 0.01, 0.1, 1.
  • coef0 - Independent term value. Recommended values: 0, 1.
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.

trainSelectorWithCrossValidation(self, victor_corpus, positive_range, folds, test_size, Cs=[0.1, 1, 10], kernels=['linear', 'rbf', 'poly', 'sigmoid'], degrees=[2], gammas=[0.01, 0.1, 1], coef0s=[0, 1], k='all')

source code 

Trains a Boundary Selector while maximizing hyper-parameters through cross-validation. It uses the TRank-at-1 as an optimization metric.

Parameters:
  • victor_corpus - Path to a training corpus in VICTOR format. For more information about the file's format, refer to the LEXenstein Manual.
  • positive_range - Maximum rank to which label 1 is assigned in the binary classification setup. Recommended value: 1.
  • folds - Number of folds to be used in cross-validation.
  • test_size - Percentage of the dataset to be used in testing. Recommended values: 0.2, 0.25, 0.33
  • Cs - Penalty parameters. Recommended values: 0.1, 1, 10.
  • kernels - Kernel functions to be used. Supported values: 'linear', 'poly', 'rbf', 'sigmoid'.
  • degrees - Degrees of the polynomial kernel. Recommended values: 2, 3.
  • gammas - Kernel coefficients. Recommended values: 0.01, 0.1, 1.
  • coef0s - Independent term values. Recommended values: 0, 1.
  • k - Number of best features to be selected through univariate feature selection. If k='all', then no feature selection is performed.
